# 대한민국 국회의원 정보 플랫폼 — 기획 문서

> **AI 교차검증 기반의, 검증 가능한 객관성을 갖춘 비영리 현직 국회의원 정보 플랫폼**

---

## 1. 개요

각종 대한민국 정치인(현직 국회의원)의 인적 사항, 과거 이력, 인물 관련 뉴스 기사 등을 한곳에서 볼 수 있는 웹사이트. 공개된 출처에 근거한 정보만을 다루며, AI를 활용해 여러 출처 간 정보의 불일치를 자동으로 탐지·표시한다.

## 2. 목적

대중에게 대한민국 정치인에 대한 **객관적 사실**을 전달하여 국민의 알권리를 보장한다.

## 3. 핵심 원칙

| 원칙 | 내용 |
| --- | --- |
| **객관성** | 주관·의견·평가를 일절 배제한다. AI는 사실을 *생성*하지 않으며, 공개된 출처 정보만 전달한다. |
| **투명성** | 코드·데이터·출처를 전부 공개하여 객관성을 *주장*이 아니라 *증명*으로 담보한다. |
| **검증** | 복수 출처의 정보 불일치를 AI가 드러낸다. 어느 정보가 옳은지에 대한 판단은 이용자에게 맡긴다. |

### 금지 사항
- 주관적 정보(개인의 생각·의견)는 전달하지 않는다.
- 세상에 공개되지 않은 정보는 다루지 않는다.
- AI가 사실을 새로 서술·요약·생성하지 않는다.

## 4. 핵심 차별점

기존의 정치인 정보 서비스(참여연대 '열려라국회', '오픈와치' 등)는 여러 출처의 정보를 **하나로 병합**해 제공한다. 본 플랫폼은 병합 대신 **출처 간 불일치를 드러낸다.**

> "AI로 데이터를 *합치는*" 것이 아니라 "AI로 데이터의 객관성을 *검증*하는" 서비스.

- AI가 핵심 가치(객관성 검증)를 직접 만들어내므로 "AI를 활용한" 프로젝트로서 정당성을 갖는다.
- AI가 사실을 생성하지 않으므로 환각 위험이 구조적으로 차단된다.
- 검증 과정과 데이터를 전부 공개하여 누구나 직접 확인할 수 있다.

## 5. 대상 범위

- **현직 국회의원 약 300명 (제22대 국회)**
- 데이터 규모가 한정적이라 품질 관리·검증이 현실적으로 가능하다.
- 향후 확장: 지방의원 → 역대 의원 (오픈소스 구조라 인수·확장 가능)

## 6. 데이터 구조 — 두 갈래

성격이 다른 데이터를 섞지 않고 분리한다. "검증이 필요한 사실"과 "참고용 외부 콘텐츠"를 구분한다.

### 6-1. 교차검증 데이터 (AI 핵심 기능)
- **대상:** 인적사항(학력·경력), 의정활동(표결·발의·위원회)
- **흐름:** 복수 출처 수집 → AI 개체정합 → 불일치 탐지·표시 → 출처 원문 링크
- **공개 범위:** 출처 메타데이터(`source_url`, `source_org`, `fetched_at`)까지 전부 공개

### 6-2. 뉴스 피드 (단순 나열)
- **대상:** 인물 관련 웹 기사·유튜브 영상
- **표시 항목:** 썸네일(임베드) · 제목 · 언론사/채널 · 일자 · 원문 링크
- **처리:** 검증하지 않음, AI 요약·생성하지 않음, 시간순 나열
- **공개 범위:** 메타데이터와 링크만 (원문·이미지 재호스팅 없음)

## 7. AI(LLM) 역할 — 3가지로 한정

| 역할 | 내용 | 필요 이유 |
| --- | --- | --- |
| **개체 정합** | 출처별로 다른 표기가 같은 항목을 가리키는지 의미적으로 판단 (예: "경제학과 졸업" ≈ "경제학 학사") | 규칙 기반으로는 의미적 동일성 판단이 어려움 |
| **불일치 분류** | 표기 차이 / 내용 충돌 / 정보 누락을 구분 | 단순 문자열 비교로는 분류 불가 |
| **자연어 질의 (RAG)** | 충돌 지점과 출처를 인용해 응답. 생성 금지, 출처 없으면 "자료 없음" 반환 | 복합 조건 질의의 접근성 향상 |

### 교차검증 동작 예시
```
출처 A (열린국회정보): "○○대학교 경제학과 1998 졸업"
출처 B (헌정회):        "○○대학교 경제학 학사"
출처 C (언론 인물DB):   "○○대학교 경영학과 졸업"
        ↓ AI가 동일 항목으로 매칭 + 불일치 탐지
[⚠ 학력 정보 불일치] 학과 표기가 출처별로 다름 (경제학과 / 경영학과)
→ 3개 출처 원문 링크 모두 제시, 판단은 이용자에게
```

## 8. 운영 모델

- **순수 재능기부** — 무수익·무후원. 어느 정당·기업에도 종속되지 않음을 구조로 보장한다.
- **전체 오픈소스** — GitHub 공개(외부 수정 불가, Pull Request 제안만 가능).
- **라이선스** — 코드: MIT 또는 Apache 2.0 / 데이터: CC BY 등 명시.
- **DB 공개 방식** — 운영 DB를 직접 노출하지 않고 정기 스냅샷(CSV/JSON)을 릴리스로 공개.
- **지속가능성** — 정적 호스팅 + 무료 티어 + 배치 갱신으로 서버비를 최소화. 오픈소스라 운영자 이탈 시에도 인수인계가 가능하다.

> **비영리 + 전체 공개는 약점이 아니라 신뢰의 근거다.** 수익·후원이 없으므로 편향의 동기가 없고, 모든 것이 공개되어 있으므로 편향 여부를 누구나 검증할 수 있다.

## 9. 기술 스택

- **Frontend:** Next.js (React 기반, 정적 생성(SSG/ISR)으로 SEO·로딩 성능 확보)
- **AI:** LLM (개체정합·불일치 분류) + RAG (질의응답)
- **데이터:** 공공 API 자동수집 + 검수, 출처 메타데이터 추적
- **주요 데이터 소스:** 열린국회정보(Open Assembly) API, 공공데이터포털, 헌정회, 선거관리위원회, 뉴스 검색 API / RSS

## 10. 정보구조 (IA)

```
메인 (검색 + 정당/지역/위원회 필터)
 ├─ 검색결과 (이름·정당·지역구)
 ├─ 정치인 상세
 │   ├─ 인적사항 (이름·나이·성별·학력·정당·직위, 불일치 배지 표시)
 │   ├─ 이력 (경력 타임라인 / 정당 이력 / 선거 이력)
 │   ├─ 의정활동 (발의법안 / 표결 / 위원회, 불일치 배지)
 │   └─ 관련뉴스 (썸네일·제목·출처·일자·링크)
 ├─ 비교 (2~3인, 2차 기능)
 └─ 질의응답 (RAG 검색·인용)
```

## 11. 객관성 강제 메커니즘

1. **출처 필수** — 출처 메타데이터 없는 데이터는 화면에 노출하지 않는다.
2. **원문 우선** — 가공 텍스트가 아니라 원문 링크로 연결한다.
3. **검수 상태 배지** — 각 정보에 "검증됨 / 검수중"을 표시해 신뢰 수준을 투명화한다.
4. **AI 답변 차단 규칙** — RAG 응답에 출처 스니펫이 없으면 추측하지 않고 "관련 자료 없음"을 반환한다.
5. **개인정보 경계** — 공개 대상은 "공인의 공적 정보"에 한정한다. 공개 기준을 문서로 명문화한다.
